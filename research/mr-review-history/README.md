# research/mr-review-history/ — the Mr. Review method record (public)

The historical record of the cold-review process: every reply from the structurally-separated
reviewer ("Mr. Review") and every locked pre-registration, kept **public and cold-reachable** so
the standing brief's falsifier rules can be re-derived from their cited sources on GitHub.

Migrated here from the old local `review-archive/` on 2026-06-09 (the disentangling: a public
research project keeps its method provenance public, not in a gitignored local bin).

## What's here

- `2026-06-0*-mr-review-feedback-*.md` — the reviewer's actual replies, dated. These are the raw
  corrections the standing brief distills into method principles.
- `2026-06-07-mr-review-stop-condition.md`, `2026-06-06-mr-review-tools-feedback.md` — topic replies.
- `2026-06-08-venture-off-preregistration.md`, `2026-06-08-voice-register-preregistration.md` — the
  locked pre-registrations (metric roles + verdict rule fixed before the data).
- `mr-review-desk/` — the reviewer's working desk (open questions, pre-test package, the metric code
  under review at the time).
- `bench-argmax-vs-sampled-2026-06-07/` — the argmax-vs-sampled bench `POSTMORTEM.md` + its 3-axis
  output (the substrate-as-motor-cortex correction). Heavy resident tarballs from that run stayed in
  the local `review-archive/` vault (uncited, too large for the public repo).

## How this relates to the rest of `research/`

- `research/runs/` — cold-verifiable **run evidence** (ledgers + recompute scripts).
- `research/review-bundle/` — the **published rounds** (what was sent to the reviewer, when).
- `research/mr-review-history/` (here) — the **replies + pre-registrations** those rounds produced.
- `research/archive.sh` — the tool that publishes a staged round and pushes it for the cold pull.

The live *method* (falsifiers, correction principles) lives in the cognition tree's standing brief,
the only document dropped into a review session; it cites into this directory.
