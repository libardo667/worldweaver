# Social Feedback And Adaptation

## Why Separate The Layers

Social feedback should not mean one undifferentiated loop that rewrites everything at once.

V6 needs three distinct layers:

- runtime adaptation
- identity growth
- offline model tuning

Keeping these separate is what prevents a humane guild from collapsing into a soft behavior-shaping machine.

## Layer 1: Runtime Adaptation

Runtime adaptation is the shortest-horizon layer.

Its job is to make bounded, reversible adjustments to:

- behavior knobs
- environment guidance

It should tune things like:

- social drive
- initiative
- repair appetite
- follow-through appetite
- caution
- mentorship receptivity

But it must do so only through narrow overlays around authored defaults.

Runtime adaptation should never:

- change model weights
- rewrite canonical soul text
- rewrite rank or branch into identity
- overwrite durable memories with one-off feedback

It is about stance and conditions, not selfhood.

## Layer 2: Identity Growth

Identity growth is the durable self-model layer.

Here, social feedback should only matter after it becomes a validated pattern.

That means:

- repeated evidence
- more than one context
- provenance back to specific feedback events or notes
- protected-core checks
- resistance to one-off contagion

The right abstraction is not “feedback writes identity.”

It is:

- feedback produces proposals
- reducers validate proposals
- only validated proposals can feed growth collapse or soul-growth paths

This keeps development legible and contestable.

## Layer 3: Offline Model Tuning

Only after enough traces have been collected should the system prepare data for model tuning.

Even then, V6 should not aim first at a single monolithic fine-tune.

The better path is:

- branch-labeled trace export
- branch-specific adapters or LoRAs
- later evaluation before deployment

Good branch families might include:

- care
- civic work
- correspondence
- research

This preserves an important separation:

- branch practice can live partly in adapters
- durable personhood remains actor-scoped and outside weights

## Feedback Sources

V6 should support both explicit and inferred feedback, but they should not have equal force.

Explicit sources:

- mentor review
- steward review
- peer commendation or correction
- quest evaluation
- review-board notes

Inferred sources:

- reciprocity and reply completion
- follow-through on queued intents and quests
- conflict versus repair patterns
- stability versus collapse of social threads
- mail completion and response quality

Explicit and inferred signals can both inform runtime adaptation.
Only repeated, validated patterns should reach identity growth.

## Protected-Core Boundaries

Some things must remain outside ordinary feedback metabolism.

Protected-core exclusions should include:

- canonical soul text
- inviolable identity anchors
- rank as a substitute for selfhood
- branch assignment as a substitute for selfhood
- raw debug traces that were never meant to become canon

Without these boundaries, the guild stops being formative and starts being invasive.

## Environment Guidance

Runtime adaptation should also shape the world around a member.

Possible guidance outputs:

- more mentor exposure
- more solo time
- lower or higher social density
- easier or harder quest band
- branch-specific invitations

This matters because humane development is not only about changing the mind.
It is also about changing the habitat.

## Trace Export For `gpt-oss-120b`

Trace export should be prepared now even if fine-tuning comes later.

The export shape should include:

- prompts
- selected context windows
- outputs
- feedback summaries
- branch metadata
- rank metadata
- outcome labels where available

It should exclude:

- canonical soul text
- protected-core-only material
- raw debug traces not needed for training
- private human data beyond disclosed governance boundaries

The goal is a future where open-weight models can learn branch practice from a humane institution without flattening identity into one giant weight update.
