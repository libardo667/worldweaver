# Isolate actor-paid inference from shared simulation and agent runtime

## Problem

Per-user key storage now exists, but billing ownership is still too implicit to trust.

- The current backend binds an actor's personal API key at the request boundary in
  `worldweaver_engine/src/api/game/action.py`,
  `worldweaver_engine/src/api/game/turn.py`, and
  `worldweaver_engine/src/api/game/story.py` via
  `worldweaver_engine/src/services/player_api_keys.py`.
- `worldweaver_engine/src/services/llm_client.py` then treats that request-scoped key
  as the default for any LLM call made during that request.
- That means a single player request can accidentally bill their key for:
  - referee / plausibility reasoning
  - world-state mutation planning
  - storylet selection or follow-up generation
  - prefetch work triggered after the action
  - any other helper inference that happens to run under the same request context
- The legacy single-call action interpreter in
  `worldweaver_engine/src/services/command_interpreter.py` mixes state reasoning and
  prose generation in one model call, making strict cost attribution impossible.
- Resident cognition lives in the shard agent runtime (`ww_agent` code mounted into the
  shard agent container), but the resident files in `shards/<city>/residents` hold the
  per-resident state, memory, and identity data. That distinction is easy to blur, and
  billing policy becomes muddy when "agent replies" are discussed without separating
  runtime code from shard-local resident state.

Without an explicit inference-ownership model, the product cannot honestly promise
"your key only pays for your slice of the narrative."

## Proposed Solution

Introduce a first-class inference ownership and billing policy that separates:

- shared world simulation work
- actor-private presentation work
- autonomous resident runtime work

This major deliberately tightens the contract beyond major 17. Storing a personal key
per actor is not enough; every inference call must declare who owns it and which key
class it is allowed to use.

### Phase 1 - Define explicit inference ownership classes

Add a small ownership model used everywhere inference is invoked:

- `platform_shared`
  - world-state reasoning, referee checks, storylet generation, prefetch, embeddings,
    bootstrap generation, shard maintenance inference
- `actor_private`
  - player-private rendering that exists only to present already-decided outcomes back
    to one actor
- `agent_runtime`
  - `ww_agent` loop cognition and any other shard-resident autonomous runtime work

Also define a small policy record carried with each call:

- `owner_type`
- `owner_id`
- `billing_scope`
- `allowed_key_sources`
- `operation`
- `trace_id`

The rule is strict:

- `platform_shared` may never use an actor key
- `agent_runtime` may never use an actor key
- `actor_private` may use an actor key only when the output is private to that actor
  and does not decide shared world state

### Phase 2 - Replace request-scoped actor key binding with call-scoped key resolution

Remove the route-level pattern where `/api/action` and related handlers bind an actor key
for the entire request.

Instead:

- route handlers resolve the authenticated actor identity only
- each inference call passes an explicit ownership policy into the LLM client layer
- the LLM client chooses a key from that policy rather than from ambient request state

This should make "which key paid for this call?" answerable from the call site alone.

### Phase 3 - Split the player turn pipeline into shared reasoning vs private rendering

Formalize the action path into two different billing domains:

- Shared, shard-paid:
  - action interpretation / intent extraction
  - plausibility and referee checks
  - state delta resolution
  - storylet / projection / follow-up selection
  - prefetch and embeddings
- Actor-private, player-paid:
  - final prose rendering of the already-resolved outcome shown back to that player

For staged action flow, the seam already exists in
`worldweaver_engine/src/services/command_interpreter.py`:

- Stage A / validation stays `platform_shared`
- Stage B / narration render becomes `actor_private`

For the legacy one-shot path:

- either disable it for BYOK actors
- or force it to remain `platform_shared`

No mixed state+prose call may be billed to an actor key.

### Phase 4 - Clarify how resident replies are billed

Document and enforce the distinction:

- `ww_agent` contains the runtime code that thinks and replies
- `shards/<city>/residents/*` contains resident-local data, not the inference engine

Policy for resident responses:

- autonomous fast/slow/mail loop thinking remains `agent_runtime`
- co-located chat or DM replies generated by those loops remain shard-paid
- if a future feature adds an explicitly player-private "render a reply just for this
  actor" path, that must be a separate `actor_private` call with no shared-world
  decision-making inside it

This prevents one player's personal key from subsidizing shared simulation or the
general social life of the city.

### Phase 5 - Add an auditable inference ledger

Persist an inference ledger row for every paid or potentially paid LLM call with:

- timestamp
- trace_id
- route
- session_id
- actor_id if present
- owner_type
- owner_id
- billing_scope
- operation
- model
- provider
- key_source_used
- prompt_tokens
- completion_tokens
- estimated_cost_usd
- success / failure

This ledger is the enforcement proof. Without it, the system has policy but not audit.

### Phase 6 - Surface the contract in product copy and diagnostics

Update the client and readiness surfaces so the product says only what is true:

- actor key pays for your private narration render, not shared world cognition
- resident autonomy remains shard-paid
- observer mode means the world still runs, but private narration render is unavailable

Add a diagnostics view or settings section that can eventually show:

- recent actor-paid calls
- model used
- estimated spend
- clear explanation of what is and is not billed to the actor

## Acceptance Criteria

- [ ] A player's personal key is never selected implicitly for an entire request
- [ ] Every inference call declares an explicit ownership / billing policy
- [ ] Shared world cognition calls (`platform_shared`) are rejected if they attempt to use an actor key
- [ ] Agent runtime calls (`agent_runtime`) are rejected if they attempt to use an actor key
- [ ] The staged action path uses shard-paid inference for state reasoning and actor-paid inference only for the final player-private prose render
- [ ] The legacy one-shot action path is either disabled for BYOK actors or guaranteed to remain shard-paid
- [ ] Prefetch, embeddings, and other background inference triggered by a player action remain shard-paid
- [ ] Autonomous resident chat and DM replies remain shard-paid unless explicitly reimplemented as a separate actor-private surface
- [ ] An inference ledger exists that can show which key source paid for each call
- [ ] Client copy and settings text describe the billing contract accurately

## Risks & Rollback

- This cuts across auth, turn orchestration, inference helpers, observability, and
  product copy. A partial rollout could easily create mixed behavior where some calls
  are policy-aware and others still inherit ambient request state.
- Disabling the legacy one-shot path for BYOK users may expose hidden dependencies on
  the old interpreter flow. Roll it out behind a feature flag if needed.
- An inference ledger introduces sensitive metadata. Do not log raw prompts or raw API
  keys; store only ownership, usage, and cost metadata.
- If the actor-private narration path fails while shared reasoning succeeded, the player
  still needs a fallback render path that does not lose the already-resolved action.
- Rollback path: keep the old global-key fallback available for all player requests,
  but do not revert to route-scoped actor key binding once call-scoped ownership is
  introduced. If billing isolation cannot be guaranteed, prefer temporarily forcing all
  player turns onto shard-paid inference over silently charging actor keys for shared
  simulation work.
