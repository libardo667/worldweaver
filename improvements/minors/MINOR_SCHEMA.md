# Minor Improvement Schema

Minor improvements are targeted, low-risk changes that improve the developer or user experience without significantly altering the core architecture. They usually touch a single file or a small cluster of closely related items.

---

## File naming

```
NN-<slug>.md        (NN = sequence number starting at 30, slug = kebab-case)
```

Example: `30-refactor-api-helpers.md`

---

## Required Sections

### Title (H1)
### Problem
### Proposed Solution
### Files Affected
### Acceptance Criteria
